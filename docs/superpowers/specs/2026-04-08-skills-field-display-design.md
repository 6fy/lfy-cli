# Skills 字段展示规范设计

## 背景

当前 lfy-cli 项目包含 5 个 skills，每个 skill 对应不同的业务功能。这些 skills 的输出直接面向业务人员，但现有的 SKILL.md 中缺乏统一的字段展示规范。

业务人员对技术性字段（如 ID、时间戳等）无法理解，因此需要统一规范字段的展示方式。

## 核心规则

| 规则 | 说明 |
|------|------|
| **技术字段默认隐藏** | `id`、`user_id`、`org_id`、`created_at`、`updated_at` 等下划线格式字段不展示 |
| **英文字段名不直接展示** | 所有字段显示时必须有中文标签 |
| **表格形式展示** | 使用中文表头 + 值的表格格式 |
| **显式字段映射** | 在每个 SKILL.md 的"典型工作流"章节中，为每个接口定义字段与中文表头的映射关系 |
| **用户要求显示时添加标注** | 当用户明确要求显示技术字段时，在表头添加"（技术字段）"标注 |

## 技术字段定义

以下字段默认视为技术字段，**不展示**给业务用户：

- 所有以 `_id` 结尾的字段（如 `user_id`、`org_id`、`pipeline_id` 等）
- 所有以 `_at` 结尾的字段（如 `created_at`、`updated_at` 等）
- 字段名仅为 `id` 的字段
- 其他 AI 判断为技术性字段的命名

**例外**：当用户**明确要求**显示时，可展示但需添加标注。

## 字段映射规范

### 放置位置

在每个 SKILL.md 的"典型工作流"章节中，为每个接口增加"展示模板"小节。

### 映射格式

```markdown
### 接口名

...（现有内容保持不变）...

**展示模板**：

| 中文表头1 | 中文表头2 | ... |
|----------|----------|------|
| 字段值1  | 字段值2  | ... |

**字段映射**：

| 原始字段 | 中文表头 | 备注 |
|---------|---------|------|
| field_a | 字段A   | 默认展示 |
| field_b | 字段B   | 默认展示 |
| id      | -       | 技术字段，默认隐藏 |
```

### 映射原则

1. **语义一致**：中文表头应与字段的业务含义一致
2. **简洁易懂**：使用业务人员能理解的词汇
3. **保持一致**：相同含义的字段在不同接口中应使用相同的中文表头

## 修改范围

### 需要修改的文件

1. `skills/lfy-customer/SKILL.md`
2. `skills/lfy-ops/SKILL.md`
3. `skills/lfy-pipeline/SKILL.md`
4. `skills/lfy-schedule/SKILL.md`
5. `skills/lfy-user/SKILL.md`

### 不需要修改的文件

- `skills/*/references/*.md`：保持 API 技术文档不变，供 AI 理解接口结构

## 示例

### lfy-user get_self 接口

**原始 API 返回**：
```json
{
  "user_id": "123",
  "user_name": "张三",
  "org_id": "456",
  "org_name": "华东销售部"
}
```

**展示模板**：

| 姓名 | 组织名称 |
|------|----------|
| 张三 | 华东销售部 |

**字段映射**：

| 原始字段 | 中文表头 | 备注 |
|---------|---------|------|
| user_name | 姓名 | 默认展示 |
| org_name | 组织名称 | 默认展示 |
| user_id | - | 技术字段，默认隐藏 |
| org_id | - | 技术字段，默认隐藏 |

### lfy-customer search 接口

假设原始返回：
```json
{
  "customer_id": "C001",
  "customer_name": "某科技公司",
  "gtm_name": "企业业务",
  "created_at": "2024-01-01 10:00:00"
}
```

**展示模板**：

| 客户名称 | GTM业务线 |
|----------|----------|
| 某科技公司 | 企业业务 |

**字段映射**：

| 原始字段 | 中文表头 | 备注 |
|---------|---------|------|
| customer_name | 客户名称 | 默认展示 |
| gtm_name | GTM业务线 | 默认展示 |
| customer_id | - | 技术字段，默认隐藏 |
| created_at | - | 技术字段，默认隐藏 |

## 实施顺序

建议按以下顺序修改：

1. `lfy-user`（用户查询，最常用）
2. `lfy-customer`（客户查询）
3. `lfy-pipeline`（商机查询）
4. `lfy-schedule`（日程任务）
5. `lfy-ops`（运营数据）

## 后续工作

1. 修改各个 SKILL.md 文件，添加展示模板
2. 确保新编写的 SKILL.md 遵循此规范
